# Архитектура URL Shortener

## Общая архитектура

```
┌─────────────────────────────────────────┐
│         API Layer (routes.py)           │  ← HTTP endpoints
├─────────────────────────────────────────┤
│      Service Layer (services/)          │  ← Бизнес-логика
├─────────────────────────────────────────┤
│   Repository Layer (repositories/)      │  ← Работа с БД
├─────────────────────────────────────────┤
│      Models Layer (models/)             │  ← SQLAlchemy модели
└─────────────────────────────────────────┘
```

---

## 📂 Структура слоев

### 1. **API Layer** (`app/api/routes.py`)
- Определяет HTTP endpoints
- Валидация входных данных через Pydantic
- Обработка HTTP запросов/ответов
- Dependency Injection для БД сессий

**Ответственность:**
- Маршрутизация запросов
- Сериализация/десериализация данных
- HTTP статус коды

---

### 2. **Service Layer** (`app/services/`)
- Бизнес-логика приложения
- Координация между репозиториями
- Валидация бизнес-правил
- Обработка ошибок

**Ответственность:**
- Генерация уникальных кодов
- Проверка дубликатов
- Подсчет статистики
- Бизнес-валидация

---

### 3. **Repository Layer** (`app/repositories/`)
- Изоляция работы с БД
- CRUD операции
- Запросы к базе данных

**Ответственность:**
- SQL запросы через SQLAlchemy
- Транзакции
- Оптимизация запросов

---

### 4. **Models Layer** (`app/models/`)
- SQLAlchemy ORM модели
- Определение схемы БД
- Связи между таблицами

**Ответственность:**
- Структура таблиц
- Индексы
- Constraints

---

## 🔄 Поток данных

### Создание короткой ссылки:

```
1. Client → POST /api/v1/shorten
              ↓
2. routes.py → Валидация через LinkCreate schema
              ↓
3. LinkService → Бизнес-логика:
                 - Проверка дубликатов
                 - Генерация/валидация кода
              ↓
4. LinkRepository → Создание записи в БД
              ↓
5. LinkService → Формирование ответа
              ↓
6. routes.py → Возврат LinkResponse
              ↓
7. Client ← JSON response
```

### Редирект:

```
1. Client → GET /{short_id}
              ↓
2. routes.py → Получение short_id
              ↓
3. LinkService → Поиск ссылки + инкремент счетчика
              ↓
4. LinkRepository → SELECT + UPDATE в БД
              ↓
5. routes.py → HTTP 307 Redirect
              ↓
6. Client ← Redirect to original_url
```

## 🗄️ Схема базы данных

### Таблица: `links`

```sql
CREATE TABLE links (
    id SERIAL PRIMARY KEY,
    original_url VARCHAR NOT NULL,
    short_id VARCHAR(10) UNIQUE NOT NULL,
    clicks INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP
);
```
